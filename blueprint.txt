Projects like UniClipboard and other LAN clipboard sync tools generally use **local network discovery + WebSockets + clipboard listeners** to provide instant syncing without a cloud server. ([GitHub][1])

# Project Name

# Goal

Create a Universal Clipboard between Android and macOS.

Requirements:

* No internet
* No accounts
* Same Wi-Fi only
* Auto-discover devices
* Copy on Android → Paste on Mac
* Copy on Mac → Paste on Android
* Near-instant sync (<500ms)
* Runs in background

---

# Tech Stack

## Mac

* Electron
* React
* Node.js
* ws
* bonjour (mDNS)
* clipboardy
* electron-store

## Android

* Kotlin
* ClipboardManager
* OkHttp WebSocket
* NSDManager (Network Service Discovery)
* Room Database

# Phase 1: Local Discovery

### Mac

Advertise service:

```txt
Service Name:
Clippr-MacBook

Service Type:
_clippr._tcp

Port:
8585
```

### Android

Scan for:

```txt
_clippr._tcp
```

Auto-connect.

No IP addresses should be entered manually.

Mac:


# Prevent Infinite Loops

Bad:

```txt
Mac
↓
Android
↓
Mac
↓
Android
```

Good:

```json
{
  "id": "abc123",
  "source": "mac-123"
}
```

Ignore:

```txt
if (message.source === myDeviceId)
    return;
```
